Below is a **clear, simple, production-ready app plan** for a **Next.js Soccer Squad Selector** — optimized for fast “vibe coding”, no backend, and localStorage only.

---

## 1. App Goal (What You’re Building)

A **lightweight web app** for your soccer team to:

* Manage squads (22 players or any number)
* Build lineups visually
* Drag & drop players between positions
* Auto-generate teams or manually edit them
* Persist everything in **browser localStorage**

No auth. No DB. No server logic.

---

## 2. Tech Stack (Minimal & Fast)

* **Next.js 14 (App Router)**
* **TypeScript**
* **Tailwind CSS**
* **@dnd-kit/core** (drag & drop)
* **localStorage** (state persistence)

---

## 3. Core Pages & Layout

## 4. Data Models (Simple)

### Player

```ts
type Player = {
  id: string
  name: string
  photo: string
  preferredPosition: Position
}
```

### Position

```ts
type Position =
  | "GK" | "LB" | "CB" | "RB"
  | "LM" | "CM" | "RM"
  | "ST"
```

### Squad Slot

```ts
type SquadSlot = {
  position: Position
  playerId?: string
}
```

### Team

```ts
type Team = {
  id: string
  name: string
  players: string[] // playerIds
}
```

## 5. Mock Data (Initial Load)

## 6. Key Features (How Each Works)

### ✅ Player Pool (Show / Hide)

* Collapsible panel
* Shows:

  * Player photo
  * Name
  * Preferred position
* Drag from pool → field

---

### ✅ Squad Board (Formation View)

* Fixed formation grid (like image you shared)
* Each position = **drop zone**
* Player card snaps into slot

Rules:

* One player per slot
* Dragging replaces existing player

---

### ✅ Drag & Drop Logic

Using **@dnd-kit**:

* Drag player:

  * From pool → squad
  * Squad → squad
  * Squad → pool
* Drop behavior:

  * Same position → swap
  * Different position → allow (no restriction)

---

### ✅ Switch Players (Same / Different Position)

* If dropped on occupied slot:

  * Swap players
* Optional UI indicator:

  * Green glow = preferred position
  * Yellow = not preferred

---

### ✅ Auto vs Manual Squad

**Manual Mode**

* User controls everything

**Auto Mode**

* One-click:

  * Fill formation based on preferred positions
  * Extra players stay in pool

Toggle switch:

```
[ Auto Squad ]  ◉─────○  [ Manual ]
```

---

### ✅ Uneven Teams UI Indicator

If:

* Team A has more players than Team B

UI:

* Red / orange border
* Badge:

```
+3 Players
```

Example:

```css
border-2 border-red-500 animate-pulse
```

---

### ✅ Add More Players

Modal:

* Name
* Position
* Optional image upload
* Save → localStorage

No limit on player count.

---

### ✅ Persistence (localStorage Only)

Saved items:

* Players list
* Squad assignments
* UI state (show/hide, mode)

Key:

```ts
localStorage.setItem("soccer-squad-state", JSON.stringify(state))
```

Auto-load on app start.
